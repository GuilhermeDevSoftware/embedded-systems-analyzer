## Cálculo do tempo de bit

Para uma transmissão em `9600 baud`:

```text
Tbit = 1 / 9600
Tbit ≈ 104,17 µs
```

As medições realizadas no Saleae apresentaram valores próximos de:

```text
104 µs
104,25 µs
```

Os resultados são compatíveis com o valor teórico esperado.

## Análise dos bytes transmitidos

### Byte `0x00`

```text
Binário: 00000000
```

O start bit e os oito bits de dados permaneceram em nível baixo, formando um único intervalo baixo de aproximadamente nove períodos de bit.

Valor teórico:

```text
9 × 104,17 µs ≈ 937,53 µs
```

Valor observado:

```text
aproximadamente 937,75 µs
```

### Byte `0xFF`

```text
Binário: 11111111
```

Somente o start bit permaneceu em nível baixo. Os oito bits de dados e o stop bit permaneceram em nível alto.

Valor observado para o start bit:

```text
aproximadamente 104 µs
```

### Byte `0x55`

```text
Binário: 01010101
Ordem transmitida: 1 0 1 0 1 0 1 0
```

A alternância frequente entre os níveis lógicos facilitou a medição individual do tempo de bit.

Valor observado:

```text
aproximadamente 104,25 µs por bit
```

### Byte `0xAA`

```text
Binário: 10101010
Ordem transmitida: 0 1 0 1 0 1 0 1
```

O start bit e o primeiro bit de dados são ambos zero. Por isso, aparecem como um único trecho baixo com duração aproximada de dois períodos de bit.

### Byte `0x41`

```text
Hexadecimal: 0x41
ASCII: A
Binário: 01000001
Ordem transmitida: 1 0 0 0 0 0 1 0
```

A captura permitiu relacionar a forma de onda elétrica com o byte hexadecimal e com o caractere ASCII correspondente.

## Observações importantes

- o baud rate permaneceu constante para todos os bytes;
- a duração de cada bit foi aproximadamente a mesma;
- o desenho da forma de onda mudou conforme o conteúdo do byte;
- bits consecutivos com o mesmo valor não produzem novas transições;
- as indicações de frequência e duty cycle do Saleae dependem da região selecionada e não representam diretamente o baud rate;
- no padrão `0x55`, a alternância a cada bit produz uma frequência visual próxima de `4,8 kHz`, pois um ciclo completo ocupa dois períodos de bit;
- o intervalo de aproximadamente `500 ms` entre os bytes foi confirmado nas capturas;
- inicialmente, a janela curta de captura separava a sequência em duas gravações; o problema foi identificado como limitação da duração da captura, e não falha da UART.

## Conhecimentos consolidados

- diferença entre baud rate, frequência da forma de onda e quantidade de transições;
- estrutura básica de um quadro UART;
- estado ocioso em nível alto;
- função do start bit em nível baixo;
- transmissão dos bits de dados com o bit menos significativo primeiro;
- função do stop bit;
- significado do formato `8N1`;
- relação entre bits, bytes, hexadecimal e ASCII;
- efeito de sequências de bits iguais sobre a forma de onda;
- uso do decodificador `Async Serial` como ferramenta de validação;
- importância de comparar a decodificação automática com a análise manual.

## Dificuldades encontradas

- a janela de captura do Saleae não registrava inicialmente todos os bytes da sequência;
- alguns bytes apresentavam formas de onda visualmente diferentes, gerando dúvida sobre a duração dos ciclos;
- arquivos auxiliares `Zone.Identifier` foram criados ao salvar imagens pelo Windows.

## Soluções aplicadas

- ajuste da sincronização e do tempo disponível para captura;
- inclusão de atraso inicial de `2 s` no firmware;
- análise individual de cada byte;
- comparação das medições com o tempo teórico de `104,17 µs`;
- identificação de que bits iguais consecutivos formam trechos contínuos;
- remoção dos arquivos `Zone.Identifier` antes do versionamento.
